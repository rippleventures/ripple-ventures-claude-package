# Calibration log

Each file captures a setup run where questions could have been better, options were missing, or the flow caused confusion.

## What gets logged

- New skills added to the plugin since last setup run (the flow needs to ask about them)
- Tech stack options that should be listed
- Questions that consistently confused users
- Wording the user pushed back on

## How to use

When a setup pass reveals something missing, copy `_template.md`, fill in, save as `YYYY-MM-DD-short-slug.md`. The next setup run reads the lessons and applies them.
